Composer

Zakaj composer?

  • Standardni problemi glede deljene uporabe kode
  • Update mehanizem - vedno najnovejša koda
  • Odprtokodni projekt z dobro dokumentacijo
  • Veliko paketov na voljo preko packagist.org
  • Delno rešuje zablode PEAR/PECL*

* tako npm kot rubygems sta daleč naprej po uporabi

Implementacija composer paketov

  • PSR je zakon! Samo 10 let prepozno,...
  • Roll your own package manager (composer-sentinel)
  • Privatni repository za pakete
  • Composer nima "hard" zahtev & omejitev

API-ji

Na kaj paziti pri APIjih?

  • Ne podvajamo kode - različni izpisi isti vir podatkov!

  • Tracking web odjemalcev (client ID, OAuth,...)
  • Statistika API klicev
  • Caching API klicev (nginx + lua + redis)

cgroups (control groups, resource mgmt)

  • control cpu time share, used cores, affinity
  • priority - cpu, disk i/o
  • accounting - see what you use
  • In the linux kernel since 2006-2007
  • Initial effort credited to Google
  • Paul Menage, Rohit Seth

Not a virtual machine

namespaces (isolation)

  • PID - Process visibility
  • NET - Network isolation
  • MNT - Mount point control
  • IPC - Inter-process communication
  • USER - User permissions and mapping
  • UTS - change hostname/domainname

Let's get specific

Resource usage: disk

Virtual Machine: ~10GB - 100GB (OS only, low data)
Docker Image: ~10MB - 100MB - 1GB (Image size, no data)
  • Docker: True dynamically expanding data (not virtual size of FS)
  • Image layers: "base" images can be reused
  • Docker: 1 Application*, VM: 1 Operating system
